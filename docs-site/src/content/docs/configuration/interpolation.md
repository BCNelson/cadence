title: Interpolation
description: ${file:...}, ${env:VAR}, ${env:VAR:-default}, $$ — when they run and how.

cadence supports two interpolation schemes in string values: `${file:...}` for reading file contents, and `${env:...}` for reading environment variables. Both run **after** layer merge, so an overlay can substitute a token from an underlying layer.

## Phases and order

Interpolation runs in two passes, in this order:

1. **`${file:PATH}`** — replaced by the contents of the file (with trailing whitespace and newlines trimmed). Relative paths resolve against the directory of the YAML file that introduced the token.
2. **`${env:VAR}`** or **`${env:VAR:-default}`** — replaced by the environment variable's value.

Files first means a `${file:...}` payload that itself contains `${env:...}` tokens will have those env tokens expanded too. The other direction doesn't work — an env value will not be re-parsed for `${file:...}` tokens.

## Forms

```yaml
server:
  uuid_salt: "${env:CADENCE_UUID_SALT}"          # required env var
  base_url:  "${env:CADENCE_BASE_URL:-http://localhost:8080}"  # with fallback

channels:
  - name: ops
    type: webhook
    url: "${file:/run/secrets/ops-webhook.url}"  # read URL from secret file
    headers:
      Authorization: "Bearer ${env:OPS_TOKEN}"

ping_keys:
  - name: cron
    key: "${file:/run/secrets/cron.key}"
```

## Errors and edge cases

- **`${env:VAR}` unset, no default** — fails startup with a precise file/line reference.
- **`${env:VAR:-default}` unset or empty** — falls back to `default` (literal string, no nesting).
- **`${file:path}` missing or unreadable** — fails startup with the resolved absolute path in the message.
- **`$$`** — a literal `$`. Two dollar signs in a YAML string survive interpolation as one `$` in the resolved string. Use this when you legitimately need a `$` next to a `{`.
- **Unrecognized scheme** — `${other:...}` is left untouched after env/file passes. Any `${...}` surviving the second pass is an error.

## Where it applies

Interpolation runs on every scalar string value after merge. It does not run on YAML map keys, on numbers, or on booleans — only string values.

## Security note

The settings file generated by the [NixOS module](/cadence/install/nixos/) lives in the world-readable Nix store. Always use `${env:...}` with an `environmentFile`, or `extraConfigFiles` for the secret-bearing YAML — never inline the secret.
