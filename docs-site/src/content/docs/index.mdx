## What cadence is

cadence is a passive monitoring daemon. Your services ping it on a schedule; if a ping doesn't arrive within `period + grace`, the check transitions to `down` and webhooks fire. On the next successful ping it returns to `up` and a recovery alert goes out.

It's wire-compatible with Healthchecks.io's ping endpoints and the read side of its v3 management API, so existing cron-job snippets and dashboards keep working unchanged.

## The defining decision

YAML is the source of truth. LevelDB persists runtime state and ping history; it never holds check definitions. Every write endpoint on the management API returns `409 Conflict` with a pointer back to the config file. Want to add a check? Edit the YAML and reload — there is no UI-driven CRUD.
