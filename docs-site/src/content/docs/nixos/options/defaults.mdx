Per-check fields override the corresponding default. The difference between "field absent" (inherit default) and "field set to `[]`" (no default applied) is preserved through the YAML emitter — setting `defaults.channels = ["ops"]` while a check sets `channels = []` means that check is genuinely silent, not inheriting `ops`.
